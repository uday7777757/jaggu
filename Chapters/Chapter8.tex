\chapter{Conclusion and Future Work}

\section{Conclusion}
This project report presented a complete machine learning based approach to customer churn prediction for a FinTech platform. The work covered the problem context, literature survey, system analysis, architecture, implementation methodology, testing strategy, and illustrative evaluation. The comparative analysis of Logistic Regression, Decision Tree, Random Forest, and XGBoost showed that ensemble models are the most suitable for the assumed structured churn dataset, with XGBoost delivering the strongest overall predictive performance.

The study also showed that churn prediction is not only a modeling problem but also a decision-support problem. By transforming churn probabilities into practical risk categories and dashboard outputs, the system becomes useful for business teams responsible for customer retention. Important features such as transaction activity, tenure, and complaints provide meaningful guidance for targeted intervention strategies.

\section{Future Work}
Several improvements can extend the present work:
\begin{itemize}
    \item integrate real-time or scheduled scoring into a production CRM workflow;
    \item add SHAP or LIME based explanations for transparent decision support;
    \item implement automatic retraining to address concept drift;
    \item expand the feature set using feedback text, channel usage, or customer sentiment data;
    \item validate business value through A/B testing of retention interventions.
\end{itemize}

In summary, the proposed churn prediction framework demonstrates the practical value of machine learning for proactive retention management in FinTech environments and provides a strong foundation for future deployment-oriented work.
