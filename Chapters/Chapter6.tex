\chapter{System Testing and Evaluation}

\section{Testing Methodology}

The testing strategy follows a four-level hierarchy: unit testing, integration testing, system testing, and model validation testing. Each level addresses different failure modes and quality attributes.

\subsection{Unit Testing}

Unit tests verify the correctness of individual functions and classes in isolation. The \texttt{pytest} framework is used with \texttt{unittest.mock} for dependency injection. Critical units tested include:

\begin{itemize}
    \item \texttt{engineer\_features()}: Verifies correct derivation of all engineered features given a minimal input DataFrame.
    \item \texttt{DataPreprocessor.transform()}: Verifies correct scaling and encoding on known inputs.
    \item \texttt{ChurnPredictor.predict()}: Verifies that probability output is in $[0, 1]$ and that label thresholding is correct.
    \item \texttt{/predict} endpoint: Verifies JSON response schema and HTTP status codes using Flask's test client.
\end{itemize}

\subsection{Integration Testing}

Integration tests verify that pipeline stages interact correctly end-to-end:

\begin{itemize}
    \item Loading raw CSV $\rightarrow$ feature engineering $\rightarrow$ preprocessing $\rightarrow$ model inference produces valid outputs.
    \item The REST API correctly serialises and deserialises JSON payloads and maps them to model inputs.
    \item The PostgreSQL connection pool handles concurrent requests without data corruption.
\end{itemize}

\subsection{System Testing}

System tests exercise the deployed Docker container under realistic conditions:

\begin{itemize}
    \item \textbf{Functional tests}: Submit 500 known customer records to \texttt{/predict} and verify that returned predictions match offline batch predictions.
    \item \textbf{Load tests}: Using the \texttt{locust} framework, 100 concurrent users submit requests for 5 minutes; the system must maintain mean response time $<$ 200 ms and P99 $<$ 500 ms.
    \item \textbf{Schema validation}: All API responses are validated against a JSON Schema definition.
\end{itemize}

\subsection{Model Validation Testing}

Statistical tests assess model reliability:

\begin{itemize}
    \item \textbf{Calibration test}: Reliability diagrams (calibration curves) verify that predicted probabilities are well-calibrated (Platt scaling applied if needed).
    \item \textbf{Stability test}: Model is retrained on five different random seeds; standard deviation of AUC-ROC across seeds must be $<$ 0.005.
    \item \textbf{Fairness audit}: Churn recall is compared across gender and geography subgroups to detect disparate impact.
\end{itemize}

\section{Evaluation Metrics}

\subsection{Primary Metrics}

Given the class imbalance (80:20 ratio), accuracy alone is an inadequate performance measure. The following metrics are used:

\begin{itemize}
    \item \textbf{Precision} ($P$): $\frac{TP}{TP + FP}$ -- fraction of predicted churners who actually churned.
    \item \textbf{Recall} ($R$): $\frac{TP}{TP + FN}$ -- fraction of actual churners correctly identified. High recall is most critical for business impact.
    \item \textbf{F1-Score}: $\frac{2PR}{P + R}$ -- harmonic mean of precision and recall.
    \item \textbf{AUC-ROC}: Area under the Receiver Operating Characteristic curve; measures rank discrimination regardless of threshold.
    \item \textbf{AUC-PR}: Area under the Precision-Recall curve; more informative than AUC-ROC under class imbalance.
    \item \textbf{Matthews Correlation Coefficient (MCC)}: $\frac{TP \cdot TN - FP \cdot FN}{\sqrt{(TP+FP)(TP+FN)(TN+FP)(TN+FN)}}$ -- balanced metric for imbalanced classes.
\end{itemize}

        & \textbf{Predicted: No Churn} & \textbf{Predicted: Churn} \\
        \hline
        \textbf{Actual: No Churn} & 1{,}481 (TN) & 112 (FP) \\
        \hline
        \textbf{Actual: Churn} & 80 (FN) & 327 (TP) \\
        \hline
    \end{tabular}
    \label{tab:confusion}
\end{table}

From Table~\ref{tab:confusion}:
\begin{itemize}
    \item True Positive Rate (Recall): $327 / (327 + 80) = 80.3\%$
    \item Precision: $327 / (327 + 112) = 74.5\%$
    \item False Positive Rate: $112 / (1{,}481 + 112) = 7.0\%$
    \item Accuracy: $(1{,}481 + 327) / 2{,}000 = 90.4\%$
\end{itemize}
