\chapter{Conclusion and Future Work}

\section{Conclusions}

This project has successfully designed, implemented, and evaluated a production-ready machine learning system for customer churn prediction in a FinTech platform context. The work addressed a clearly articulated problem: existing rule-based and simple statistical approaches to churn management achieve inadequate recall on imbalanced data, lack interpretability mechanisms required by financial regulators, and are not architected for integration into live customer engagement platforms.

The key conclusions drawn from this project are as follows:

\subsection{Conclusion 1: XGBoost is the Superior Algorithm for FinTech Churn Prediction}

Among the six algorithms evaluated---Logistic Regression, Decision Tree, Random Forest, SVM, ANN, and XGBoost---the Bayesian-tuned XGBoost classifier consistently achieved the best performance across all evaluation metrics. On the held-out test set, it achieved an AUC-ROC of 0.923, recall of 80.3\%, precision of 74.5\%, and F1-Score of 0.773. This confirms findings from comparative studies in the literature \cite{chen2016xgboost, lu2020xgboost} and demonstrates the effectiveness of gradient boosting for structured tabular financial data.

\subsection{Conclusion 2: SMOTE is the Most Effective Imbalance Handling Strategy}

The 80:20 class imbalance present in the dataset caused a naive XGBoost model to achieve recall of only 51.2\% despite overall accuracy of 91.2\%. SMOTE resampling raised recall to 80.3\% while simultaneously improving AUC-ROC from 0.912 to 0.923, outperforming class-weight adjustment, random over-sampling, and the SMOTE-ENN variant. This validates the recommendation in the literature \cite{chawla2002smote} and underscores the necessity of systematic imbalance handling.

\subsection{Conclusion 3: Feature Engineering Significantly Enhances Predictive Power}

The engineered features, particularly \texttt{InactiveWithBalance}, \texttt{GermanyFemale}, \texttt{BalanceSalaryRatio}, and \texttt{AgeGroup}, contributed substantially to model performance. Random Forest baseline AUC-ROC improved from 0.872 (raw features only) to 0.902 (with all engineered features), a gain of 3 percentage points attributable purely to feature engineering. This confirms that domain knowledge expressed as features remains competitive with purely algorithmic advances \cite{coussement2008churn}.

\subsection{Conclusion 4: SHAP Provides Actionable and Regulatorily Compliant Interpretability}

SHAP TreeExplainer values provided consistent, stable, and domain-interpretable feature attributions. Age, account balance, number of products, activity status, and geography were identified as the top five churn drivers, aligning with domain expectations. Individual SHAP force plots enable customer service agents to understand the specific factors driving a given customer's churn risk score, satisfying the explainability requirements of financial regulators \cite{lundberg2018treeshap}.

\subsection{Conclusion 5: The System Architecture is Production-Ready}

The Flask REST microservice, packaged as a Docker container and integrated with PostgreSQL for persistent storage, demonstrated sub-200ms mean response latency under 100 concurrent users in load testing. The modular design, with a versioned model registry and independent preprocessing pipeline, supports scheduled retraining and A/B testing of model versions without service interruption.

\subsection{Conclusion 6: Significant Business Value is Achievable}

The business impact analysis estimates that a retention programme powered by the proposed model would generate approximately \$27 million in net annual revenue value for a FinTech platform with 500{,}000 customers, representing a 15x return on the campaign investment. This quantifies the direct business case for investment in ML-based churn prediction infrastructure.

\section{Summary of Contributions}

The principal contributions of this project are:

\begin{enumerate}
    \item A comprehensive, reproducible experimental benchmark comparing six ML algorithms under consistent preprocessing and evaluation conditions on a FinTech-representative churn dataset.

    \item A domain-informed feature engineering module comprising ten engineered features derived from raw demographic and account data, with documented performance improvement attributable to each feature group.

    \item The first integrated study (to the authors' knowledge) applying Bayesian hyperparameter optimisation, SMOTE resampling, SHAP interpretability, and decision threshold tuning \textit{together} on this dataset, with explicit evaluation of each component's marginal contribution.

    \item A production-ready microservice architecture with Docker containerisation, REST API design, and PostgreSQL integration, bridging the gap between academic ML prototypes and operational deployments.

    \item A quantified business impact analysis framework that translates model performance metrics into estimated revenue impact and ROI, enabling data-driven business case construction.
\end{enumerate}

\section{Future Work}

Several directions for future research and system enhancement are identified:

\subsection{Temporal Modelling with Survival Analysis}

The current system treats churn as a static binary event. A more nuanced approach would model \textit{time to churn} using survival analysis (e.g., Cox Proportional Hazards model, Accelerated Failure Time models) or recurrent neural networks (LSTM, GRU) trained on longitudinal transaction sequences. This would enable the system to predict not only \textit{whether} a customer will churn but \textit{when}, allowing more precisely timed retention interventions \cite{verbeke2012new}.

\subsection{Uplift Modelling for Optimal Intervention Targeting}

The current system predicts churn probability but does not estimate the incremental effect of a retention intervention on a given customer (treatment effect). Uplift modelling (causal machine learning) using meta-learners (S-Learner, T-Learner, X-Learner) would enable the system to identify customers who are (a) likely to churn and (b) genuinely persuadable by the retention offer, maximising campaign ROI \cite{gutierrez2017causal}.

\subsection{Deep Learning with Transaction Sequences}

Raw transactional logs (payment amounts, merchants, timestamps) contain rich behavioural signals not captured by aggregated tabular features. Transformer-based sequence models or Graph Neural Networks applied to customer transaction graphs could substantially improve recall for behavioural churn patterns \cite{wang2022transformer}.

\subsection{Federated Learning for Privacy Preservation}

Deploying the system across multiple banking partners requires sharing customer data or model gradients across institutional boundaries. Federated learning frameworks (e.g., PySyft, Flower) would enable collaborative model training without centralising sensitive customer data, satisfying GDPR data minimisation requirements \cite{mcmahan2017federated}.

\subsection{Automated ML Pipeline and MLOps}

The current system requires manual retraining triggers. Integration with an MLOps platform (MLflow, Kubeflow, or SageMaker Pipelines) would automate the full model lifecycle: drift detection triggering retraining, automated evaluation against a champion model, and zero-downtime deployment via canary or shadow modes.

\subsection{Multi-Class Churn Taxonomy}

Distinguishing between different types of churn (complete account closure vs. product downgrade vs. reduced engagement) would enable more targeted interventions. A multi-class or hierarchical classification approach could learn separate models for each churn type from labelled customer outcome records.

\subsection{Real-Time Streaming Integration}

The current batch-and-serve architecture could be extended to a real-time streaming pipeline using Apache Kafka and Apache Flink, enabling instant churn risk re-scoring as customer transactions occur. This would allow interventions within the same customer session (e.g., targeted in-app offers triggered by a customer browsing a competitor's product page).

\section{Final Remarks}

Customer churn prediction represents one of the highest-value applications of machine learning in the FinTech sector. This project demonstrates that a well-engineered ML pipeline---combining systematic feature engineering, principled imbalance handling, rigorous multi-metric evaluation, model-agnostic interpretability, and production-grade deployment architecture---can deliver both statistically strong and business-relevant churn prediction capabilities. The methodological framework developed here is transferable to other customer retention challenges across the broader financial services industry.
