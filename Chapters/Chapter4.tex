\chapter{System Design}

\section{System Architecture}
\label{sec:architecture}

The proposed system follows a layered microservices architecture that decouples data ingestion,
model training, and inference serving. Figure~\ref{fig:architecture} illustrates the high-level
architecture, showing the flow from raw data sources through ETL, preprocessing, feature
engineering, model training, and finally to inference API delivery.

\begin{figure}[H]
\centering
\begin{tikzpicture}[
  node distance=0.8cm and 1.4cm,
  box/.style={draw, rectangle, rounded corners, minimum width=3.5cm, minimum height=0.75cm,
              font=\small, align=center, fill=blue!10},
  arrow/.style={->, thick}
]
  \node[box, fill=green!15] (src)  {Data Sources (DB/Logs/CRM)};
  \node[box, below=of src]  (etl)  {ETL Pipeline (Python/Airflow)};

\caption{High-Level System Architecture for the Churn Prediction Platform}
\label{fig:architecture}
\end{figure}

The architecture is designed with the following principles:

\begin{itemize}
    \item \textbf{Separation of concerns}: Training and inference paths are independent; model
          updates do not require downtime of the serving API.
    \item \textbf{Reproducibility}: All pipeline parameters, feature transformations, and model
          artefacts are versioned and tracked in MLflow, enabling full experiment reproducibility.
    \item \textbf{Scalability}: The FastAPI inference service is stateless and horizontally
          scalable via container orchestration (Kubernetes).
    \item \textbf{Observability}: Prediction distributions, model drift indicators, and API
          latency metrics are continuously monitored via Grafana dashboards.
\end{itemize}

The deployment architecture features:

\begin{itemize}
    \item \textbf{Load balancer}: Distributes incoming scoring requests across multiple API pods.
    \item \textbf{FastAPI pods}: Stateless inference containers that load the champion model
          from the MLflow registry at startup.
    \item \textbf{Redis cache}: Stores pre-computed feature vectors for recently scored customers,
          reducing database load for repeat requests.
    \item \textbf{PostgreSQL}: Persists scoring results, SHAP values, and audit logs.
    \item \textbf{MLflow model registry}: Version-controlled model artefacts with staging and
          production environments.
\end{itemize}
