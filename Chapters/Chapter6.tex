\chapter{System Testing \& Evaluation}

\section{Testing Approach}
Testing is carried out at multiple levels to validate both the analytical pipeline and the system workflow:
\begin{itemize}
    \item \textbf{Unit testing}: validates isolated functions such as missing-value treatment, encoding, and prediction output formatting.
    \item \textbf{Integration testing}: verifies complete execution from data input to risk report generation.
    \item \textbf{Model evaluation}: measures predictive performance on unseen test data.
    \item \textbf{Usability testing}: ensures dashboard outputs are understandable for business users.
\end{itemize}

\section{Evaluation Metrics}
Because churn data is typically imbalanced, model selection cannot rely on accuracy alone. This project therefore uses multiple metrics, with special emphasis on the churn class \cite{systematicreview,bhattacharjee2026}.

\begin{table}[H]
\centering
\caption{Evaluation Metrics}
\begin{tabular}{|p{3cm}|p{8.5cm}|}
\hline
\textbf{Metric} & \textbf{Definition} \\ \hline
Accuracy & $(TP + TN) / (TP + TN + FP + FN)$ \\ \hline
Precision & $TP / (TP + FP)$ \\ \hline
Recall & $TP / (TP + FN)$ \\ \hline
F1-Score & $2 \times \frac{Precision \times Recall}{Precision + Recall}$ \\ \hline
ROC-AUC & Area under the ROC curve \\ \hline
\end{tabular}
\label{tab:evaluation_metrics}
\end{table}
