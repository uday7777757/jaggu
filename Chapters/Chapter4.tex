\chapter{System Design}

\section{System Architecture}

The proposed churn prediction system follows a layered microservice architecture comprising five principal layers: Data Ingestion, Preprocessing \& Feature Engineering, Model Training \& Selection, Model Serving, and Client Integration. Figure~\ref{fig:architecture} presents the overall system architecture.

\begin{figure}[H]
    \centering
    \begin{tikzpicture}[node distance=1.2cm, auto,
        layer/.style={draw, fill=blue!15, rectangle, rounded corners,
                      minimum width=12cm, minimum height=0.9cm, text centered},
        arrow/.style={->, thick}]
        \node[layer, fill=gray!20] (client) {Client Layer: CRM / Marketing Automation / Dashboard};
        \node[layer, fill=blue!20, below of=client] (api) {API Layer: Flask REST Microservice (Docker Container)};
        \node[layer, fill=green!20, below of=api] (model) {Model Layer: XGBoost Pipeline + SHAP Explainer};
        \node[layer, fill=yellow!20, below of=model] (feat) {Feature Engineering Layer: scikit-learn Pipeline};
        \node[layer, fill=orange!20, below of=feat] (data) {Data Layer: PostgreSQL / CSV / Streaming Ingestion};
        \draw[arrow] (client) -- (api);
        \draw[arrow] (api) -- (model);
        \draw[arrow] (model) -- (feat);
        \draw[arrow] (feat) -- (data);
    \end{tikzpicture}
    \caption{Overall System Architecture}
    \label{fig:architecture}
\end{figure}

The architecture is designed for modularity: each layer is independently deployable and replaceable. The Flask API exposes a \texttt{/predict} endpoint that accepts a JSON payload containing customer feature values and returns a churn probability, a binary prediction, and SHAP-based feature attributions.

\subsection{Activity Diagram}

The activity diagram documents the model training workflow (Listing~\ref{lst:actdiag}).

\begin{lstlisting}[language=,caption={Activity Diagram -- Model Training Workflow -- Mermaid Diagram Code},label={lst:actdiag}]
flowchart TD
    A([Start]) --> B[Load raw CSV dataset]
    B --> C[Drop identifier columns]
    C --> D[Encode categorical features]
    D --> E[Engineer derived features]
    E --> F[Stratified train/test split 80/20]
    F --> G[Apply SMOTE to training set]
    G --> H[Standard scale continuous features]
    H --> I{For each algorithm}
    I --> J[Train with 5-fold cross-validation]
    J --> K[Bayesian hyperparameter search]
    K --> L[Evaluate on validation folds]
    L --> M{More algorithms?}
    M -->|Yes| I
    M -->|No| N[Select best model by AUC-ROC]
    N --> O[Final evaluation on held-out test set]
    O --> P[Compute SHAP values]
    P --> Q[Serialise pipeline to disk]
    Q --> R([End])
\end{lstlisting}
