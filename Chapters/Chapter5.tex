\section{Data Loading and Preprocessing}

The data loading module reads customer records from CSV or Parquet files into a pandas DataFrame,
performs schema validation, and hands the validated frame to the preprocessing pipeline.

\begin{lstlisting}[language=Python, caption={Data loading and preprocessing snippet}]
import pandas as pd
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.impute import SimpleImputer
from sklearn.compose import ColumnTransformer
from category_encoders import TargetEncoder

def build_preprocessor(num_cols, cat_low, cat_high):
    num_pipeline = Pipeline([
        ('imputer', SimpleImputer(strategy='median')),
        ('scaler',  StandardScaler()),
    ])
    cat_low_pipeline = Pipeline([
        ('imputer', SimpleImputer(strategy='most_frequent')),
        ('encoder', pd.get_dummies),  # one-hot
    ])
    cat_high_pipeline = Pipeline([
        ('imputer', SimpleImputer(strategy='most_frequent')),
        ('encoder', TargetEncoder()),  # target encoding
    ])
    return ColumnTransformer([
        ('num',      num_pipeline,      num_cols),
        ('cat_low',  cat_low_pipeline,  cat_low),
        ('cat_high', cat_high_pipeline, cat_high),
    ])
\end{lstlisting}

The \texttt{ColumnTransformer} ensures that numeric, low-cardinality categorical, and
high-cardinality categorical columns are processed by their respective sub-pipelines in a single
fit-transform call, preventing any information leakage between the training and test subsets.

\section{Feature Engineering Implementation}

The feature engineering module constructs the 47 derived features described in Chapter~3. The
implementation processes each customer's transaction history using a sliding-window approach.

\begin{lstlisting}[language=Python, caption={RFM feature construction}]
from datetime import timedelta

def compute_rfm(df: pd.DataFrame,
                ref_date: pd.Timestamp,
                window_days: int = 90) -> pd.DataFrame:
    cutoff = ref_date - timedelta(days=window_days)
    recent = df[df['txn_date'] >= cutoff]
    rfm = recent.groupby('customer_id').agg(
        recency  = ('txn_date',  lambda x: (ref_date - x.max()).days),
        frequency= ('txn_id',    'count'),
        monetary = ('txn_amount','sum'),
    ).reset_index()
    return rfm

def compute_rolling_stats(df: pd.DataFrame,
                          col: str,
                          windows: list = [30, 60, 90]) -> pd.DataFrame:
    result = {}
    for w in windows:
        rolled = df.groupby('customer_id')[col].rolling(w, min_periods=1)
        result[f'{col}_mean_{w}d'] = rolled.mean().reset_index(0, drop=True)
        result[f'{col}_std_{w}d']  = rolled.std().reset_index(0, drop=True)
    return pd.DataFrame(result)
\end{lstlisting}

XGBoost achieves the highest ROC-AUC (0.9147) and F1-score (0.8312) across the five folds and

The SMOTE oversampling (with Tomek link cleaning) is applied to the training fold within each
cross-validation iteration, not to the entire training set before splitting, to prevent optimistic
bias in the reported metrics.

\begin{lstlisting}[language=Python, caption={SMOTE within cross-validation}]
from imblearn.pipeline import Pipeline as ImbPipeline
from imblearn.combine import SMOTETomek
from xgboost import XGBClassifier
from sklearn.model_selection import StratifiedKFold, cross_validate

pipeline = ImbPipeline([
    ('preprocess',   build_preprocessor(num_cols, cat_low, cat_high)),
    ('resample',     SMOTETomek(random_state=42)),
    ('classifier',   XGBClassifier(random_state=42, eval_metric='logloss')),
])

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_validate(
    pipeline, X_train, y_train, cv=cv,
    scoring=['roc_auc', 'f1', 'recall', 'precision'],
    return_train_score=True,
)
\end{lstlisting}

0.9247 (tuned parameters), representing a gain of 0.010 AUC points.

\begin{lstlisting}[language=Python, caption={Optuna objective function for XGBoost tuning}]
import optuna
from sklearn.model_selection import cross_val_score

def objective(trial):
    params = {
        'n_estimators':      trial.suggest_int('n_estimators', 100, 1000, step=50),
        'max_depth':         trial.suggest_int('max_depth', 3, 10),
        'learning_rate':     trial.suggest_float('learning_rate', 0.005, 0.3, log=True),
        'subsample':         trial.suggest_float('subsample', 0.5, 1.0),
        'colsample_bytree':  trial.suggest_float('colsample_bytree', 0.4, 1.0),
        'min_child_weight':  trial.suggest_int('min_child_weight', 1, 10),
        'gamma':             trial.suggest_float('gamma', 0, 5),
        'reg_alpha':         trial.suggest_float('reg_alpha', 1e-8, 1.0, log=True),
        'reg_lambda':        trial.suggest_float('reg_lambda', 1, 5),
    }
    model = XGBClassifier(**params, random_state=42, eval_metric='logloss')
    pipe  = ImbPipeline([
        ('pre',      build_preprocessor(num_cols, cat_low, cat_high)),
        ('smote',    SMOTETomek(random_state=42)),
        ('clf',      model),
    ])
    score = cross_val_score(pipe, X_train, y_train, cv=5,
                            scoring='roc_auc').mean()
    return score

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=100)
\end{lstlisting}

\subsection{REST API Design}

The FastAPI application exposes three endpoints:

\begin{itemize}
    \item \texttt{GET /health}: Returns service status, model version, and uptime.
    \item \texttt{POST /score/batch}: Accepts a JSON array of customer identifiers and returns
          churn probability scores and top-5 SHAP attributions for each customer.
    \item \texttt{POST /score/single}: Accepts a full feature vector for a single customer and
          returns an immediate synchronous prediction.
\end{itemize}

The batch scoring endpoint uses an asynchronous worker pool (via \texttt{asyncio} and \texttt{concurrent.futures})
to parallelise feature retrieval and model inference for large request batches.
